## Summary

**Knowledge-Based Voting Architecture:**

1. **Observation Layer**: BGP events observed and stored in knowledge base
2. **Storage Layer**: Hybrid in-memory + persistent disk storage
3. **Consensus Layer**: Peer voting based on matching observations
4. **Blockchain Layer**: Validated transactions committed to immutable chain

**Key Files:**
- `p2p_transaction_pool.py` - Voting logic + knowledge base
- `observer_main.py` - BGP observation + knowledge base updates
- `blockchain_data/state/knowledge_base.json` - Persistent storage

**Storage Location:** `blockchain_data/state/knowledge_base.json`
