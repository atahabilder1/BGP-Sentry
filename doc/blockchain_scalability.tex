\section{Architecture Overview}

BGP-Sentry uses a \textbf{per-node independent blockchain} architecture. Each RPKI
validator maintains its own append-only chain of blocks linked by SHA-256 hashes.
There is no shared primary chain or global ledger. This design was chosen for:
\begin{itemize}
    \item \textbf{Horizontal scalability}: No global bottleneck; each node writes independently.
    \item \textbf{Fault isolation}: A corrupted or malicious node's chain does not affect others.
    \item \textbf{Parallel processing}: All nodes can commit blocks concurrently without
          distributed locking.
\end{itemize}

Blocks are replicated to a gossip subset of peers. When a replicated block conflicts
with the local chain tip (different \texttt{previous\_hash}), a \textbf{fork-merge block}
is created that incorporates novel transactions from the peer's block.

\section{Conclusions}

\begin{enumerate}
    \item \textbf{Chain integrity}: 100\% valid across 8,838+ independent chains.
    \item \textbf{Fork resolution}: 100\% --- all forks merged successfully, even at
          1.98M forks (2030/hop2).
    \item \textbf{P2P reliability}: Zero message drops across 4.97M+ messages.
    \item \textbf{Scalability}: Runtime scales sublinearly with node count (6$\times$ nodes
          $\to$ 1.5$\times$ runtime).
    \item \textbf{TPS}: Network throughput reaches 20.6 TPS at the largest configuration,
          bounded by observation rate rather than consensus capacity.
    \item \textbf{Architecture validated}: The per-node independent blockchain design
          eliminates global bottlenecks while maintaining data integrity through
          fork-merge blocks and gossip replication.
    \item \textbf{Hop scaling}: Infrastructure handles hop=3 loads (450K forks, 563K
          messages) without degradation, demonstrating headroom beyond the recommended
          hop=2 configuration.
\end{enumerate}

\section{Discussion Summary}

The per-node independent blockchain architecture achieves 100\% chain integrity across all 8 runs---8,838 individual chains verified with every SHA-256 hash and previous-hash linkage correct. Fork resolution is perfect: all forks detected (up to 1.98 million in the largest configuration) were successfully resolved through merge blocks. The P2P message bus delivered 4.97 million messages across all runs with zero drops, validating the lock-free tokio mpsc channel design with 4,096-message per-node buffers. Runtime scales sublinearly with network size---going from 445 to 2,671 RPKI nodes (6$\times$) only increased wall-clock time from 36 to 54 minutes (1.5$\times$), demonstrating effective parallelism. Network TPS reaches 20.6 at the largest hop=2 configuration (2030 dataset, 1,371 nodes), bounded by observation rate rather than consensus capacity. The architecture eliminates global bottlenecks entirely: each node writes to its own chain without contention, and gossip replication ensures cross-network data propagation through fork-merge blocks.
